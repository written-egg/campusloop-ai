## 2. 已完成功能

### 收藏

- 登录后显示“我的收藏”导航，未登录时隐藏。
- 商品详情支持收藏、取消收藏、处理中禁用和失败提示。
- 未登录点击收藏会跳转登录页；自己的商品不能收藏。
- “我的收藏”展示商品图片、名称、价格、成色、卖家、校区、状态和收藏时间。
- 收藏列表包含加载、空数据和错误状态；已售出或已下架商品仍可查看。
- 收藏结果来自 `GET/POST/DELETE` 收藏 API，没有使用 `localStorage` 模拟。

### 站内消息

- 登录后显示“消息”导航，未登录时隐藏。
- 商品详情支持“联系卖家”；未登录会跳转登录页，自己的商品不能联系。
- 消息页采用会话列表和消息线程双栏布局，显示商品、对方昵称、最后消息和未读数量。
- 支持按商品与对方账号加载消息、标记收到的未读消息、发送新消息。
- 发送前去除首尾空格；空消息和超过 1000 字符的消息被拦截；发送中按钮禁用。
- 发送成功后使用后端返回消息更新线程，并重新获取会话列表。
- 390px 手机布局采用单栏，会话列表进入线程后显示“返回会话”。

### AI 历史

- 登录后显示“AI 记录”导航，未登录时隐藏。
- 支持全部、智能估价、风险评估三种后端筛选。
- 展示报告类型、模型来源、评分、时间和关联商品。
- 估价报告展示建议价、价格区间和估价依据。
- 风险报告展示风险等级、结论和主要建议。
- 报告可展开查看完整 JSON 结果。
- `deepseek` 显示为“DeepSeek”，`local-fallback` 显示为“本地回退”。
- 现有估价和风险评估请求已携带当前登录用户标识及认证头，使 SQL Server 报告可归属当前账号。

## 6. 需要 A 配合确认

`GET /api/products` 的 SQL Server 商品映射当前没有返回 `sellerId`，但首次从商品详情创建会话必须向 `POST /api/messages` 提供 `receiverId`。前端已支持以下两种可用情况：

1. 商品对象包含 `sellerId` 时可以直接联系卖家。
2. 已有会话时可以从会话中的 `peerId` 恢复联系人。

对于“SQL Server 返回的商品且买家从未联系过卖家”的情况，前端会明确提示缺少卖家账号标识，不会猜测接收人或伪造发送成功。建议 A 在公开商品返回中补充只读的 `sellerId`，然后再做一次首次联系卖家的真实 SQL 验收。
